# civil-accountability — 설계 원칙

목표: **건축물 시공의 책임추적 (audit/책임 규명)**.
사고·하자 발생 시 "누가 어떤 결정을 어떤 근거로 내렸고, 그 시점에 법정 배치가 충족됐나"를 결정적으로 회수한다.

이 팩은 **현장 데이터가 아니라 도메인 모델(틀 + 규칙)**이다. 어느 현장이든 자기 데이터를 이 틀에 채워 판정받는다.

---

## 핵심 축 3개

책임추적의 진입·판정은 항상 이 3축의 교차에서 일어난다.

1. **장소 (Location)** — 건물 > 층 > 실/구역 > 부위. 사고가 발견되는 1차 진입점.
2. **부재/공종 (BuildingComponent; 공종축은 WorkRecord.work_kind enum)** — 콘크리트·철근·외벽 등. 원인 분석·cross-cut.
3. **시간 (Time)** — 모든 결정·기록·선임이 시점을 갖는다. 사고 시점 기준 판정의 축.

장소 × 부재 교차 = 책임 단위 ("3층 외벽의 12/15 타설 콘크리트").
거기에 시간이 붙어 "언제 누가 결정" 이 완성된다.

---

## 확정 원칙 1 — event_date 는 모든 결정·기록 노드의 1급 속성

> 결정(Decision)·기록(Log)·검수·선임 등 **시점성 있는 모든 노드는 `event_date` 를 required 로 갖는다.**

근거: 책임추적에 들어오는 모든 원천 문서(시공일지·감리일지·검측서·자재검수일보·선임계·사고보고서)는 **시간이 무조건 박혀 있다.** 따라서 시간은 추가 부담이 아니라 원천 데이터에 이미 존재 → 가짜 필드 위험 없음.

적용:
- 단일 시점 사건 → `event_date` (date 또는 datetime)
- 유효구간을 갖는 것(선임 등) → `start_date` (required) + `end_date` (nullable, null = 현재까지 유효)
- 현장(Project) → `construction_start_date` + `construction_end_date` (착공계·계약서)
- 판정 → `assessment_date` (사고일 등 조회 기준시점)

판정 시점(assessment_date) 기준으로 "그 시점에 유효했던 것"만 카운트한다.
예: 선임은 했으나 사고 전 해임된 경우(end_date < assessment_date)는 공백으로 포착.

## 확정 원칙 2 — 책임은 무한 재귀가 아니라 "결정별 법정 책임주체"에서 끊긴다

도급 사슬(원청→하청→재하청)을 통째로 모델링하지 않는다.
각 결정마다 **법이 지정한 책임 역할**이 있고 거기서 책임이 확정된다.

RACI 대신 (법적 책임이 1명이 아닐 수 있으므로) 다음으로 표현:
- **실행자(performer)** N명 — 실제 작업자
- **법정 책임자(accountable)** — 결정 종류가 정하는 자격자. **인원은 법정 배치규칙이 정함** (1명 고정 아님)
- **입회/검토(witness)** — 감리·안전관리자 입회
- **보고대상(informed)** — 발주청 등

조직 위계는 **옵션 2 깊이**: 사람마다 `소속 조직 + 도급단계(원청/하청/차수)` 속성만. 사슬 전체 트리는 안 만든다. 중대재해처벌법 경영책임자 등 법이 책임을 전가하는 지점만 edge.

---

## 확정 원칙 3 — Reference 규칙(rules/)이 팩의 심장

배치·자격 요건은 raw 법령 텍스트가 아니라 **machine-checkable 규칙**으로 직렬화한다.
`actual >= required` 계산이 되어야 typed graph 의 의미가 산다 (RAG 로는 불가).

규칙 = 현장 데이터를 비교당하는 기준:
```
[Reference 규칙]  공사금액 850억 → 안전관리자 2명 필요 (별표3)
[현장 데이터]     실제 유효 선임 1명 (사고 당일)
[판정]            understaffed, shortfall 1  → 중대재해처벌법 책임 가중
```

규칙 파일은 반드시 `authority`(근거 법령·별표·개정일·출처)를 갖고, 미검증 부분은 `pending` 에 정직 표기. 추측 금지 — 원문 대조한 것만.

현재 직렬화:
- `rules/safety_manager_staffing.yaml` — 산안법 별표3 안전관리자 선임 (검증 완료, 시간축 반영)

후속 직렬화 예정:
- 감리(건설사업관리기술인) 배치기준 — 건진법 별표 (공사비+연면적 → 등급별 인원)
- 별표4 안전관리자 자격 — qualification_ref 정의

---

## 데이터 두 층 (이전 OpenCRAB 분석에서 확인한 분리에 용도 부여)

| 층 | 내용 | 위치 |
|---|---|---|
| Reference (이 팩) | 타입 틀 + 법정 배치/자격 규칙 | `types/`, `rules/` |
| Project (현장) | 실제 현장의 위치·부재·결정·선임 인스턴스 | 현장이 채움 (팩 아님) |

판정 = Project 를 Reference 규칙에 대조.

---

## 범위 (좁힘 확정)

- 시설: **건축물** 고정 (교량·터널·댐 등 토목 제외 — 기존 팩의 토목 KDS/KCS/진단 70% 는 범위 밖)
- 목표: **B (책임추적)** 단일. 컴플라이언스 체커·작성어시스턴트 등은 범위 밖
- 첫 vertical slice: **안전관리자 선임 적정성** (시점 기준)

## 갱신 — requirement_shape 5종 (rule 템플릿 확장)

모든 배치규칙은 동일 envelope 위에서 `requirement_shape` 로 산출 형태만 달리한다. evaluator 가 shape로 dispatch:

| shape | 산출 | 예 |
|---|---|---|
| simple_count | 규모 → 요구 인원수 (+increment 공식) | 안전관리자, 보건관리자 |
| threshold_binary | 임계값↑ → 선임의무 1명 | 안전보건(총괄)관리책임자, 조정자 |
| qualification | 규모 → 수용 자격 프로파일(등급·자격증·경력 OR), count | 현장대리인, 책임CM |
| graded_slots | 규모 → 등급별 인원 슬롯 (등급 greedy 매칭) | 품질관리자 |
| field_slots | 조건 → 분야별 인원 (총괄+상주 시 분야별) | 공사감리자 |

판정 출력: compliant / understaffed(인원부족) / unqualified(자격·등급미달) / not_required(의무없음).
모든 shape 공통: assessment_date 시점 **유효 선임**(start≤t≤end)만 카운트 → 시간축 일관 적용.

## 직렬화 안 한 것 (정직)

- 주택법 주택건설공사 감리원 별표1, 건진법 CM 상주/기술지원 인원: **법령에 고정 인원표 부재**(인·월 산출공식·비율). 날조 금지 — rule 화 보류. 건축물 일반은 공사감리자(건축법)가 커버.

## 갱신 — 5개 책임 차원으로 확장

배치(staffing) 단일 차원에서 출발해, 책임추적을 5개 차원으로 확장. 모두 동일 철학(시점 기준·법령 근거·계산 판정):

| 차원 | rule family | 판정 단위 | 시간축 적용 |
|---|---|---|---|
| 인적 배치 | rules/ (requirement_shape ×5) | 현장×역할 | 사고 당일 유효 선임 |
| 의사결정 권한 | (trail 교차검증) | 결정 | 결정 시점 유효 선임 |
| 작업 조건 | condition_rules/ (mode obligation/prohibition; value_bounds·lookup_bounds·measures) | 작업(WorkRecord) | 작업일 조건 |
| 선행요건 | prerequisite_rules/ | 작업 | 선행 검측 시점 ≤ 작업일 |
| 문서 제출 | document_rules/ | 현장 | 착공 전 제출 |

report.py 가 사고 1건으로 5차원을 일괄 평가 → 책임 가중 지점 단일 보고서. 각 위반은 근거 법령 + (배치의 경우) 책임주체로 귀속.

판정 방식 요약:
- actual ≥ required (배치 인원)
- 자격 프로파일 매칭 (배치 자격)
- 등급/분야 슬롯 greedy 매칭 (품질·감리)
- 조건 → 의무 조치 충족 / 금지 조건 위반 (작업조건)
- 범주별 기준 lookup (부재별 강도·지반별 기울기)
- 선행 결정 존재·시점·결과 (선행요건)
- 제출 대상 판정 → 제출 기록 (문서)
